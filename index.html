Showcase de Código

Limpo. Tipado. Expressivo.

Uma arquitetura feita para desenvolvedores que valorizam as melhores práticas.

<?php
declare(strict_types=1);

use FastRoute\RouteCollector;
use App\Controllers\UsuarioController;
use App\Middlewares\JwtAuthMiddleware;

// Roteamento agrupado com Middlewares nativos
return function(RouteCollector $r) {
    $r->addGroup('/api/usuarios', function (RouteCollector $r) {
        // Rotas protegidas exigem o JwtAuthMiddleware
        $r->addRoute('GET', '', [UsuarioController::class, 'index', JwtAuthMiddleware::class]);
        $r->addRoute('POST', '', [UsuarioController::class, 'store']);
        $r->addRoute('PUT', '/{id:\d+}', [UsuarioController::class, 'update', JwtAuthMiddleware::class]);
    });
};
namespace App\Controllers;

use App\Models\UsuarioModel;
use App\Views\UsuarioResource;
use App\Exceptions\BadRequestException;
use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;

class UsuarioController extends Controller  
{  
    // Injeção de Dependências via PHP-DI
    public function __construct(
        protected UsuarioModel $model,
        protected UsuarioResource $resource
    ) {}

    public function store(ServerRequestInterface $request): ResponseInterface
    {  
        $data = $this->getBody($request);
        $errors = $this->validate($data, ['nome' => 'required', 'email' => 'email|required']);

        if (!empty($errors)) {
            throw new BadRequestException('Dados inválidos.');  
        }

        // Senhas são formatadas com PASSWORD_ARGON2ID
        $usuario = $this->model->create([
            'nome'  => htmlspecialchars((string) $data['nome']),
            'email' => $data['email'],
            'senha' => password_hash($data['senha'], PASSWORD_ARGON2ID),
        ]);

        return $this->resource->created($this->resource->item($usuario->toArray()));
    }  
}
namespace App\Views;

class UsuarioResource extends Resource  
{  
    /**
     * Transforma um item antes de retornar em JSON.
     * Semelhante ao Laravel API Resources.
     */
    protected function transform(array $item): array
    {
        return [
            'id'         => $item['id'],
            'nome'       => $item['nome'],
            'email'      => $item['email'],
            'criado_em'  => date('d/m/Y H:i', strtotime($item['created_at'])),
            // Observe que a 'senha' nunca é retornada na view!
        ];
    }
}

Fundação Robusta

Construído sob Excelência Técnica

O Parrot PHP não reinventa a roda; ele integra de forma magistral os pacotes mais sólidos da comunidade PHP com as mais restritas normas de segurança.

PHP 8.4+ Strict

Desenvolvido com tipagem rigorosa (`declare(strict_types=1);`), union types, e construtores promovidos para garantir código à prova de falhas.

Segurança OWASP Global

Proteção by-design contra SQL Injection, senhas em Argon2id nativo, XSS mitigation e Rate Limits integrados via pipeline de middlewares.

Autenticação JWT Stateful

Gerenciamento seguro de sessão que vai além. Login via cookies HttpOnly e sistema de controle de Blacklist (Tokens Revogados) embutido no banco.

Eloquent ORM Standalone

Configure e manipule com MariaDB/MySQL usando o ORM mais amado (Laravel Database). Migrations e Seeds incluídos no core (database/scripts).

FastRoute & PSR-7/15

Roteamento nativo usando nikic/fast-route, processando requisições sob os moldes rigorosos das interfaces HTTP do PHP-FIG (Nyholm).

Test-Driven (PHPUnit)

Já entregue com ampla cobertura de testes de integração, testando efetivamente endpoints em banco de dados real em vez de SQLite local.

Injeção de Dependências

Contêiner configurado usando PHP-DI. Classes, Controllers e Middlewares recebem suas dependências automaticamente através de construtores.

Padrão em Respostas JSON

Uma camada dedicada de Views (Resources) para isolar completamente a lógica de transformação dos retornos e mascarar dados sensíveis da API.

Pronto para Produção

Alta compatibilidade com FrankenPHP e Caddy Server, garantindo que sua aplicação receba milhares de conexões em pouquíssimos milissegundos.

Visão Estrutural

Arquitetura Limpa e Desacoplada

Parrot PHP encoraja a adoção de Single Responsibility e separação de camadas. Nenhuma requisição atinge o Controller de forma insegura.

  • 1

    Interceptação Perfeita (Middlewares)

    Rate Limiting, CORS, Proteção JWT e Headers de Segurança agem como um escudo rigoroso inspirado na documentação OWASP, processados antes da sua lógica de negócio.

  • 2

    Lógica Isolada (Controllers & Models)

    Todo acesso a dados usa o consagrado Eloquent ORM sem risco de SQL injection. Controllers apenas coordenam o fluxo.

  • 3

    Retorno Padronizado (Resources)

    A camada final `src/Views` garante que propriedades indesejadas (como hashes de senhas e tokens ocultos) nunca vazem no output da API.

Estrutura de Diretórios 

parrot-php/
├── config/        # DI Container, Rotas
├── database/      # Migrations CLI, Seeds
├── public/        # front-controller index.php
└── src/
    ├── Controllers/ # Handlers (Ponto de entrada)
    ├── Core/        # Núcleo da Aplicação PSR
    ├── Exceptions/  # Exceções HTTP Customizadas
    ├── Middlewares/ # Camadas de Processamento HTTP
    ├── Models/      # Camada de Dados (Eloquent)
    └── Views/       # View layer (Classes Resource Json)
Dependências Geridas via PHP-DI 100% PSR-7 / PSR-15
